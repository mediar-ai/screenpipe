# 入门指南

<MotionDiv>

screenpipe 提供多种安装方法，以满足用户需求和技术水平。无论您是希望使用 CLI 的开发人员，还是希望使用即插即用桌面应用程序的用户，screenpipe 都提供灵活的选项，让您快速入门。

对于非技术用户或希望快速入门的用户，我们建议获取[桌面应用程序](https://screenpi.pe/onboarding)。

</MotionDiv>

<MotionDiv delay={0.3}>

## 1. 安装选项

### CLI 安装

- **适合技术用户**: screenpipe 可以通过命令行界面（CLI）安装和使用。
- **说明**:
  1. 克隆仓库。
  2. 安装依赖项（确保已安装 Rust、Cargo 和其他所需工具）。
  3. 构建项目并运行 CLI。

</MotionDiv>

<MotionDiv delay={0.5}>

### 桌面应用程序

- **付费桌面应用程序**: 提供优先支持、1 年更新和独家功能。用户可以直接从官方网站下载并安装桌面应用程序。
- **免费桌面应用程序**: 开源并可自行构建。安装需要从源代码构建或向仓库发送 PR。两种选项都提供具有所有关键功能的完整桌面应用程序。

</MotionDiv>

<MotionDiv delay={0.7}>

### 手动构建

对于希望完全控制或自定义的用户，可以从仓库手动构建应用程序。此方法需要 Rust 和其他必要的依赖项。请按照提供的构建指南编译应用程序。

</MotionDiv>

## macOS 安装

<MotionDiv delay={1.2}>

### **选项 1: 从源代码构建（适合高级用户）**

1. 首先安装 Rust 和所有必要的依赖项：
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   brew install pkg-config ffmpeg jq tesseract cmake wget
   ```

2. 安装 Deno CLI（构建所需）：
   ```bash
   curl -fsSL https://deno.land/install.sh | sh
   ```

3. 克隆 screenpipe 仓库：
   ```bash
   git clone https://github.com/mediar-ai/screenpipe
   cd screenpipe
   ```

4. 构建项目：
   ```bash
   cargo build --release --features metal
   ```

5. 最后，运行 screenpipe：
   ```bash
   ./target/release/screenpipe
   ```

6. 构建桌面应用程序：

在项目根目录的 `.vscode/settings.json` 中添加以下内容：

```json
{
   "rust-analyzer.cargo.features": [
      "metal",
      "pipes"
   ],
   "rust-analyzer.server.extraEnv": {
      "DYLD_LIBRARY_PATH": "${workspaceFolder}/screenpipe-vision/lib:${env:DYLD_LIBRARY_PATH}",
      "SCREENPIPE_APP_DEV": "true"
   },
   "rust-analyzer.cargo.extraEnv": {
      "DYLD_LIBRARY_PATH": "${workspaceFolder}/screenpipe-vision/lib:${env:DYLD_LIBRARY_PATH}",
      "SCREENPIPE_APP_DEV": "true"
   },
   "terminal.integrated.env.osx": {
      "DYLD_LIBRARY_PATH": "${workspaceFolder}/screenpipe-vision/lib:${env:DYLD_LIBRARY_PATH}",
      "SCREENPIPE_APP_DEV": "true"
   }
}
```

```bash
cd screenpipe-app-tauri
bun install
bun scripts/pre_build.js # <- 这很重要，用于将 CLI 复制到应用程序中
bun tauri build
```

[需要帮助？在 GitHub 上打开一个 issue。](https://github.com/mediar-ai/screenpipe/issues/new)

### **选项 2: Homebrew 安装（仅安装 CLI）**

1. 要使用 Homebrew 安装 screenpipe，只需执行以下命令：
   ```bash
   brew tap mediar-ai/screenpipe https://github.com/mediar-ai/screenpipe.git
   brew install screenpipe
   ```

2. 安装后，使用以下命令运行 screenpipe：
   ```bash
   screenpipe
   ```

3. 可选标志：
   ```bash
   screenpipe -h
   ```

如果您需要其他配置，例如在调试模式下运行或切换到 WebRTC 进行音频处理，可以参考官方 [screenpipe CLI 文档](https://github.com/mediar-ai/screenpipe)。

</MotionDiv>

## Windows 安装（实验性）

<MotionDiv delay={1.7}>

### **在 Windows 上安装的步骤**

> 这是对 Windows 构建的实验性支持。假设您已经安装了 CUDA 工具包并将 CUDA_PATH 设置为我的 CUDA v12.6 文件夹。
> 将 `V:\projects` 和 `V:\packages` 替换为您自己的文件夹。

如果这对您不起作用，请[打开一个 issue](https://github.com/mediar-ai/screenpipe/issues/new?assignees=&labels=dislike&template=dislike.yml&title=windows+install+screenpipe+didnt+work)或获取预构建的[桌面应用程序](https://screenpi.pe)

安装 7zip: https://www.7-zip.org/download.html

```batch
choco install pkgconfiglite deno
winget install -e --id GnuWin32.UnZip
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
cd V:\projects
git clone https://github.com/mediar-ai/screenpipe
cd V:\packages
git clone https://github.com/microsoft/vcpkg.git
cd vcpkg
bootstrap-vcpkg.bat -disableMetrics
vcpkg.exe integrate install --disable-metrics
vcpkg.exe install ffmpeg

SET PKG_CONFIG_PATH=V:\packages\vcpkg\packages\ffmpeg_x64-windows\lib\pkgconfig
SET VCPKG_ROOT=V:\packages\vcpkg
SET LIBCLANG_PATH=C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Tools\Llvm\x64\bin
cd V:\projects\screenpipe

cargo build --release --features cuda

cd screenpipe-app-tauri
bun install
bun scripts\pre_build.js # (如果您遇到 wget 不检索错误，请使用 '-O' 标志) <- 这很重要，用于将 CLI 复制到应用程序中
bun tauri build
```

</MotionDiv>

## Linux 安装

### **选项 1: 从源代码构建**

1. 使用以下命令安装依赖项：
   ```bash
   sudo apt-get install -y libavformat-dev libavfilter-dev libavdevice-dev ffmpeg tesseract-ocr libtesseract-dev
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```

2. 安装 Deno CLI（构建所需）：
   ```bash
   curl -fsSL https://deno.land/install.sh | sh
   ```

3. 克隆仓库并构建应用程序：
   ```bash
   git clone https://github.com/mediar-ai/screenpipe
   cd screenpipe
   cargo build --release
   ```

4. 要运行应用程序，请使用：
   ```bash
   ./target/release/screenpipe
   ```

5. 构建桌面应用程序：

```bash
cd screenpipe-app-tauri
bun install
bun scripts/pre_build.js # <- 这很重要，用于将 CLI 复制到应用程序中
bun tauri build 
```

</MotionDiv>

<MotionDiv delay={2.3}>

### **选项 2: 通过 Nix 安装 CLI**

screenpipe 可以轻松安装在基于 Nix 的系统上。请按照以下说明进行操作：

1. 在 Nix shell 中运行：
   ```bash
   nix-shell -p screen-pipe
   ```

2. 或将其添加到您的系统包中：
   ```nix
   environment.systemPackages = with pkgs; [
     screen-pipe
   ];
   ```

编辑配置后，使用以下命令重建系统：
   ```bash
   sudo nixos-rebuild switch
   ```
